不能在一直helloworld了

写这种 感受还是去石墨文档吧
这里其实就是学一下搭博客的技术

也算是折腾了一下
主要是回忆了一下前年装的centos6
特别强调一下版本是因为 2020年底这个系统已经停止维护了。
所以当我打开Virtual box 再打开这个系统时 又查了查yum 这个命令（和ubantu不同apt-get 命令）这两个好像属于两个linux的分支，前一个隶属于redhat系偏向服务器端，后一个偏向用户端debian什么的）好像也就这个rpm（软件包管理器不同）

还看到了这个镜像上运行的redis。。。
难道又要回归ubantu了么？不知道，当时还创建了好几个镜像要搭建分布式，网络配置什么的都搞好了呢，甚至都已经学会了hdfs命令了哈哈哈

说回这个静态博客本身吧，这里强调的静态应该是不和服务端产生交互，修改完直接提交，只在网页端导航（nav），所以和tomcat那种服务器交互的也不太一样。
而github内置的引擎Jekyll似乎就是做这件事。而这个网站用hexo（也是一种静态引擎）但是因为如上的原因（centos6不维护，yum install ruby下不下来，ruby是Jekyll的前置条件），干脆还是先用windows powershell吧，又由于Jekyll在windows上好像很麻烦，干脆就换hexo吧（不知什么时候已经装好了nodejs（hexo的前置条件）环境，所以直接npm install hexo-cli）。在之后hexo init（初始化）hexo s（启动）这篇文章就产出了。

– 后记
当然powershell里用不了vim，一些linux命令也不太相同似乎，mkdir一个目录结果在之前的git下会冲突。直接告诉我换个地方创建就行了。。。果然。
